# Audit Project Skills

Review the `.skills/` directory as if you were a coding agent starting work on
this repository.

Identify knowledge that would require unnecessary repository exploration.

Look for repeated searches such as:

* finding how to run the project
* locating important modules
* discovering architectural boundaries
* inferring coding conventions
* reconstructing common implementation patterns
* understanding domain concepts
* locating integrations
* debugging recurring failures

If this knowledge is not already captured, add or improve the appropriate
skill.

Remove stale information.

Consolidate duplicate information.

Split overly large files if they have become difficult to navigate.

The objective is to maximize information density while minimizing maintenance
effort.

A future coding agent should be able to answer common project questions by
consulting the skills before searching the repository.

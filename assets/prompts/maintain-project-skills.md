# Maintain Project Skills

Update the `.skills/` directory to reflect the changes made during this task.

Do not rewrite everything.

Only update files affected by the current work.

Look for:

* new commands
* changed workflows
* new architectural components
* changed directory structure
* new conventions
* new dependencies
* new business concepts
* new troubleshooting information
* architectural decisions introduced during this task

Remove obsolete information.

Merge duplicate information.

Prefer editing existing sections over creating new ones.

If no meaningful knowledge has changed, leave the skills untouched.

The goal is to keep the knowledge base accurate while minimizing churn.
